### Gotcha

<!-- lore:019e0b2e-c9e4-79ba-a112-9ff664aa3a39 -->
* **Deleting generated extension surfaces requires contract updates**: Deleting legacy extension surfaces requires updating contracts/generators together: \`package.json\` files\[], \`scripts/smoke.sh\`, \`.claude-plugin/marketplace.json\`, \`.codex/config.toml\`, docs graph/version checks, and legacy bundle/manifest builders. Keep old generator commands as compatibility no-ops or they may recreate stale \`.codex-plugin\`, Bundle/Gemini, or root department dirs. Do not treat scaffolded \`docs/departments/\*\` user workspaces as deleted extension surfaces.

<!-- lore:019e0b25-a0a5-7a1f-a4f2-9f27e5c74638 -->
* **Do not commit local\_cache**: local\_cache/ is generated cache/model data and should not be committed. If it appears untracked, add or keep it in .gitignore rather than reviewing it as source changes.

<!-- lore:019e0b25-a060-7d9b-b92f-afd355e5d3b7 -->
* **Do not delete Codex plugin manifest**: Canonical Codex integration intentionally has no \`.codex-plugin/\`; do not restore it unless reverting to the old manifest\_path model. \`.codex/config.toml\` must not contain \`manifest\_path = ".codex-plugin/plugin.json"\`; discovery should rely on \`.codex/INSTALL.md\`, \`.codex/config.toml\`, \`integrations/codex/\`, and canonical agents/skills/commands. Dangling manifest paths break integration.
